AI、ML、DL 三層架構與分類
L11101 · 基礎必背
必考
▶
概念說明範例
AI人工智慧，最大範疇，讓機器模擬人類智慧行為語音助理、自動駕駛
ML機器學習，AI 子集，從資料中自動學習規律推薦系統、詐欺偵測
DL深度學習，ML 子集，使用多層神經網路影像辨識、語音辨識
⬥ 弱 AI vs 強 AI vs 超級 AI
弱 AI（窄 AI）：只能執行特定任務，目前所有 AI 都屬此類（ChatGPT、AlphaGo）
強 AI：具備人類等級的通用智慧，目前尚未實現
超級 AI：超越人類智慧，理論概念，尚未存在
⬥ 四種學習方式（情境題必考）
監督式：有標籤資料 → 分類/迴歸問題（垃圾郵件偵測）
非監督式：無標籤資料 → 分群/降維（客群分析）
半監督式：少量有標籤 + 大量無標籤資料
強化學習：透過獎懲機制學習 → 動態決策（下圍棋、自動駕駛）

AI 治理、倫理與法規
L11102 · L11203 · 115 年新增高頻
必考
▶
⬥ 台灣《人工智慧基本法》（115 年新增考點）
生效日：2026 年 1 月 14 日
中央主管機關：國家科學及技術委員會（國科會）
核心：促進 AI 創新發展，同時保障人民基本權利與公共利益
特色：不針對特定技術立法，採風險管理框架
⬥ 歐盟 AI Act 四大風險等級（必背）
🔴 不可接受風險：禁用。社會信用評分、大規模即時生物特徵辨識
🟠 高風險：嚴格規管。招募 AI、信貸審核、醫療、執法
🟡 有限風險：透明標示要求。聊天機器人（需告知用戶是 AI）
🟢 最低風險：幾乎無規範。垃圾郵件過濾、推薦系統

⭐ 考點：員工臉部辨識打卡 = 高風險（生物特徵辨識用於就業）
⬥ Human-AI 協作三模式（高頻考點）
Human-in-the-loop：人在決策流程「內部」，每一步都需人工審批
Human-on-the-loop：AI 自動執行，人類在旁「持續監督」可隨時介入
Human-over-the-loop：人設定目標規則，AI 全自動執行，人在最高層「宏觀管控」
倫理原則說明
公平性不因性別、種族等歧視，模型不產生偏見
可解釋性AI 決策過程能被人類理解（XAI）
透明性揭露 AI 使用情況，標示 AI 生成內容
問責性明確 AI 決策的責任歸屬
隱私保護個人資料，遵守個資法 / GDPR
⬥ AI 黑箱 → 問責性問題（115-2 考點）
AI 系統判斷過程不透明，發生錯誤時無法釐清責任歸屬（開發商？部署企業？使用者？）
HR 比喻：AI 做了一個人事決定，但沒人能解釋為什麼——出了問題找不到人負責
解法：事前建立責任鏈（誰開發、誰部署、誰使用各負什麼責任）+ 選用可解釋 AI
⬥ AI 對齊問題（Alignment Problem）
確保 AI 追求的目標符合人類真正的意圖，而非字面上設定的目標
例：設定「最大化用戶參與時間」→ AI 學會推送煽動性內容（達成目標，但違背初衷）
口訣：AI做的是「你真正要的」，不是「你字面說的」——兩者之間的落差就是對齊問題

資料處理、清洗與特徵工程
L11201 · L11202 · 陷阱題重災區
必考
▶
資料生命週期：收集 → 清洗 → 前處理 → 特徵工程 → 建模 → 評估 → 部署
資料類型：結構化（表格/CSV）、非結構化（圖像/文字/音訊）、半結構化（JSON/XML）
缺失值處理：刪除（資料量充足時）、均值/中位數填補（數值型）、眾數填補（類別型）
異常值偵測：Z-score（超過 ±3 標準差）、IQR 四分位距法
One-hot Encoding：類別型特徵轉為 0/1 數值（如「紅/藍/綠」→ 三個 0/1 欄）
標準化（Standardization）：轉為均值 0、標準差 1，適合常態分布資料
正規化（Normalization）：壓縮至 [0,1] 區間，適合數值範圍差異大的資料
⬥ ⚠ 資料洩漏（Data Leakage）— 高頻陷阱題
正確做法：先分割訓練集/測試集 → 只用訓練集計算均值/標準差 → 再套用到測試集
錯誤做法：用全部資料計算標準化參數（讓測試集資訊「洩漏」進訓練過程）
記憶口訣：「先切後算，只用訓練」
⬥ 類別不平衡處理（SMOTE）
問題：正負樣本比例極度不均，模型偏向多數類
解法：SMOTE（合成少數過採樣技術）、調整損失函數權重、資料增強

機器學習模型、評估與調優
L11301 · L11302 · 情境題核心
必考
▶
⬥ 情境題解題公式（反覆背誦）
看到「預測數值/金額/數量」 → 迴歸（線性迴歸、決策樹迴歸）
看到「預測是/否、類別」 → 分類（邏輯迴歸、決策樹、SVM）
看到「找相似群體、無標籤」 → 分群（K-Means、階層式分群）
看到「動態決策、遊戲、機器人」 → 強化學習
模型類型適用場景
線性迴歸迴歸預測連續數值（房價、銷售額）
邏輯迴歸分類二元分類（是/否流失、垃圾郵件）
決策樹分類/迴歸可解釋性高，規則清晰的情境
隨機森林分類/迴歸多棵決策樹集成，準確率高
K-Means分群客群分析、文件分類（無標籤）
SVM分類高維度資料，二元分類
⬥ 過擬合 vs 欠擬合（必背）
過擬合（Overfitting）：訓練準確率高 but 測試準確率低（記憶了訓練資料雜訊）
解法：Dropout、L1/L2 正則化、Early Stopping、增加訓練資料多樣性

欠擬合（Underfitting）：訓練和測試準確率都低（模型太簡單）
解法：增加模型複雜度、更多特徵、更長訓練時間
評估指標公式概念適用
Accuracy正確預測數 / 總數一般情況
Precision預測為正 → 真的為正的比例降低誤報
Recall真正為正 → 被正確預測的比例降低漏報
F1-ScorePrecision × Recall 的調和平均不平衡資料
AUC-ROC曲線下面積，越接近 1 越好二元分類整體效能
⬥ 可解釋 AI（XAI）對象選擇
SHAP：向工程師解釋，找問題根源（特徵重要性數值）
反事實解釋：向客戶解釋「你要改什麼才能過關」（如：貸款被拒的改善建議）
Saliency Map：影像模型，顯示模型「看哪裡」做決策
PDP（部分依賴圖）：呈現某特徵對整體預測的平均影響趨勢（全局）；無法解釋單一案例，那要用 LIME
⬥ 混淆矩陣：FP vs FN 代價判斷（高頻情境題）
False Positive（假陽性）：預測有問題，實際沒有 → 代價：多做一次檢查
False Negative（假陰性）：預測沒問題，實際有 → 代價：錯過真正的問題

醫療癌症篩檢 → FN 代價最高（漏掉真正的癌症患者）→ 最大化 Recall（召回率）
詐欺偵測 → FP 代價高（騷擾正常客戶）→ 最大化 Precision（精確率）
口訣：漏報傷人命用Recall；誤報傷體驗用Precision
⬥ 學習曲線（Learning Curve）診斷
X軸=訓練資料量，Y軸=訓練/驗證誤差
兩條線都高（誤差大） → 高偏差（欠擬合）→ 加更多資料沒用，要讓模型更複雜
訓練低、驗證高（差距大） → 高變異數（過擬合）→ 加更多資料可能有幫助
HR 比喻：員工體檢報告——同時有多項異常才要換治療方向，不能只看一個指標
⬥ Q-Learning（強化學習核心概念）
Q值：在狀態 s 下採取行動 a，預計能獲得的長期累積獎勵
Agent 的目標：學習每個（狀態, 行動）組合的 Q 值，每次選 Q 值最高的行動
HR 比喻：每個「處境+決策」組合，都有一個「長期績效分數」，AI 學會選分數最高的那個
⬥ mAP（物件偵測）/ BLEU（文字生成）評估指標
mAP（mean Average Precision）：物件偵測專用，同時衡量類別辨識準確度 + 邊界框定位準確性；YOLO 等模型的標準指標
BLEU 分數：文字生成/翻譯專用，計算生成文字與標準答案的 n-gram 重疊程度；高分不代表語意正確，只代表用詞相似
口訣：看圖找物件用mAP；AI寫文章/翻譯用BLEU

深度學習與神經網路架構
L11304 · L11305
中頻
▶
架構特點適用
CNN卷積層提取局部特徵，對位移不變影像辨識、電腦視覺
RNN有記憶的序列模型，但有梯度消失問題時間序列、早期 NLP
LSTM改良 RNN，加入記憶閘門解決梯度消失長文本、序列預測
Transformer自注意力機制，可並行運算LLM 基礎架構（GPT/BERT）
激活函數：ReLU（最常用，計算快）、Sigmoid（輸出 0~1，用於二元分類輸出層）、tanh（輸出 -1~1）
遷移學習（Transfer Learning）：使用已在大型資料集訓練好的模型，在新任務上微調，節省時間和資源
邊緣 AI（Edge AI）：在終端裝置（手機/IoT）推論，優點：低延遲、資料不傳雲端（隱私保護）；缺點：運算資源有限
⬥ ResNet 殘差網路（Residual Network）
引入跳躍連接（Skip Connection）：把某層輸出直接加到幾層後的輸出，讓梯度走「捷徑」不需通過每一層
解決深層網路訓練困難（梯度消失）問題，讓 100+ 層超深網路成為可能
HR 比喻：不用逐級彙報，可以直接跳過中間主管把資訊送到最高層——資訊不會在層層傳遞中消失
⬥ Transformer 位置編碼（Positional Encoding）
Self-Attention 本身對詞的位置不敏感（不管順序，計算結果相同）
位置編碼為每個 Token 加入「我在第幾個位置」的資訊，讓模型理解詞序
口訣：沒有位置編碼，Transformer 不知道「我愛你」和「你愛我」有什麼差別
⬥ 多模態學習 / 自編碼器 / GNN
多模態學習：AI 同時處理文字+圖像+音訊（如 GPT-4o 看圖說話）；比單一資料類型更接近人類理解方式
自編碼器（Autoencoder）：學習資料的壓縮表示，主要用於無監督異常偵測（學習正常資料→重建誤差大 = 異常）
圖神經網路（GNN）：處理具複雜關係結構的資料（社交網路、分子結構、知識圖譜），傳統 CNN/RNN 無法直接處理這類非規則結構
⬥ 語音識別系統架構
聲學模型（Acoustic Model）：把語音的聲學特徵（聲音）對應到音素/字——負責「聽聲辨字」
語言模型（Language Model）：根據語言規律確認字組成合理的句子——負責「字成句子」
兩者協同工作才能準確識別語音；聲學模型是語音識別的核心環節

進階 ML：集成學習、超參數與模型壓縮
114/115 年高頻考點 · 200 題多次出現
必考
▶
⬥ 偏差-變異數權衡（Bias-Variance Tradeoff）
高偏差（High Bias）= 欠擬合，模型太簡單，訓練和測試都差
高變異數（High Variance）= 過擬合，模型太複雜，訓練好測試差
增加複雜度 → 偏差↓、變異數↑（兩者無法同時最小化）
⬥ 集成學習：Bagging vs Boosting
Bagging（並行）：多棵樹同時訓練，投票決定結果。代表：隨機森林
Boosting（串行）：後面的模型專注修正前一個的錯誤。代表：XGBoost
口訣：Bagging 並行大家一起；Boosting 串行互相補救
超參數搜尋方法說明特點
Grid Search窮舉所有組合穩定但慢
Random Search隨機採樣組合較快，效果不錯
Bayesian Optimization根據過去結果智慧選下一個最省資源、最聰明
⬥ 模型壓縮四方法（邊緣部署必備）
剪枝（Pruning）：移除不重要的連接/神經元，讓網路變瘦
量化（Quantization）：降低數值精度（float32→int8），縮小模型大小
知識蒸餾（Knowledge Distillation）：大模型（教師）把知識傳給小模型（學生）
低秩分解（Low-rank Factorization）：用低秩矩陣近似原始權重矩陣
⬥ 自監督學習（Self-Supervised Learning）
從資料本身自動生成訓練標籤，不需要人工標註
例：GPT 的「預測下一個詞」、BERT 的「填入被遮蔽的詞」
→ 大型語言模型能用海量未標註文字訓練的根本原因
⬥ 自編碼器（Autoencoder）異常偵測
訓練方式：只用正常資料訓練，讓模型學會「壓縮→重建正常資料」
偵測方式：新資料送入後，重建誤差大 = 異常（模型不熟悉這種資料，重建失敗）
適合場景：無法預先標記所有異常類型（如工廠設備故障、網路入侵）
HR 比喻：新員工做了一件很不像這個部門風格的事，就是異常信號
⬥ PDP（部分依賴圖）的限制 / CRF 鑑別式模型
PDP 限制：只呈現整體趨勢（全局），無法解釋單一案例的原因——要解釋某筆資料為何被這樣預測，要用 LIME（局部）
CRF（條件隨機場）：鑑別式模型，直接學習「給定輸入序列→最可能的標籤序列」，常用於 NER（命名實體辨識）等序列標注任務；與生成式模型相對，不生成資料只做判斷

進階資料處理、NLP 與降維技術
114/115 年陷阱題重點 · 涵蓋多個常考名詞
必考
▶
⬥ 特殊特徵編碼
循環特徵（sin/cos encoding）：星期幾、時間、月份等循環資料，用正弦/餘弦轉換保留循環鄰近關係（如週日和週一在向量空間中很近）
多類別（Multi-class）：每個樣本只屬於一個類別（從多個選一）
多標籤（Multi-label）：每個樣本可同時屬於多個類別（一篇文章可同時標「AI+醫療+倫理」）
⬥ 降維技術比較
PCA：線性降維，找變異最大的方向，適合一般特徵壓縮
t-SNE：非線性降維，專門用於資料視覺化，把高維資料壓成 2D/3D 圖，相似資料會聚在一起
口訣：壓縮特徵用 PCA；畫出來看用 t-SNE
NLP 基礎概念說明
Tokenization把文字切成模型能讀的基本單位（詞/子詞/字元），NLP 第一步
詞嵌入（Word Embedding）把詞轉為數值向量，保留語意關係（「國王」-「男」+「女」≈「女王」）
情感分析（Sentiment Analysis）判斷文字的正/負/中性情感，用於品牌監控、客戶回饋分析
NER 命名實體辨識從文字中找出人名、地名、機構名、日期等特定實體
⬥ 資料品質四維度（必背）
完整性：資料有沒有缺失值
一致性：不同來源的資料定義是否相同
準確性：資料是否正確反映真實情況
時效性：資料是否還是最新的
口訣：完整一致，準確即時

MLOps、資料漂移與 AI 系統運維
115 年考題新增 · 中高頻考點
中頻
▶
⬥ MLOps 是什麼
整合 ML 開發和 IT 運維的實踐，自動化 ML 生命週期（訓練→部署→監控→更新）
HR 比喻：就像 HR 系統要定期更新員工資料、監控使用狀況、發現問題就修正——MLOps 讓 AI 模型也能這樣持續運作
⬥ 資料漂移 vs 概念漂移（高頻陷阱）
資料漂移（Data Drift）：輸入資料的分布改變（疫情後消費者行為大變）→ 模型預測失準
概念漂移（Concept Drift）：輸入和輸出的「關係」改變（以前加班=高績效，現在不一定）→ 更難偵測
兩者都需要重新訓練模型，概念漂移更隱蔽
MLOps 組件功能
資料管線（Data Pipeline）自動化資料收集→清洗→特徵工程的整個流程
特徵商店（Feature Store）集中管理特徵定義，各團隊共用，避免重複計算
模型版本控制記錄每個版本的資料/代碼/超參數，出問題可回滾
A/B 測試真實環境中對比新舊模型，用業務指標決定採用哪個
殘餘風險採取所有緩解措施後仍存在的剩餘風險，不可能歸零
⬥ 聯邦學習（Federated Learning）
各機構（如不同醫院）在自己的伺服器本地訓練，只上傳模型參數（不是原始資料）到中央伺服器聚合
→ 解決醫療、金融等敏感資料「不能集中」的合規問題
口訣：資料不動，模型動
⬥ 冷啟動問題（Cold Start）
推薦系統中，新用戶或新商品因為沒有歷史互動資料，系統無法給出有效推薦
解法：問用戶初始偏好、用熱門商品推薦、用內容特徵推薦
⬥ 密度估計異常偵測
建立資料分布模型：密度高的區域 = 正常；密度低的區域 = 異常
無監督式，不需要異常標籤，適合信用卡詐欺、設備故障等場景
口訣：人跡罕至的地方就是異常
⬥ 對抗式攻擊（Adversarial Attack）
在圖片上加入人眼看不出的微小雜訊，AI 就把熊貓認成長臂猿
對自動駕駛等安全關鍵應用是嚴重威脅；防禦方法：對抗式訓練、輸入過濾
口訣：人看不出來，AI卻被騙了

生成式 AI 原理與 LLM 技術
L12101 · L12102
必考
▶
⬥ 鑑別式 AI vs 生成式 AI
鑑別式 AI（Discriminative）：判斷輸入屬於哪個類別（垃圾郵件偵測、圖像分類）
生成式 AI（Generative）：創造全新內容（文字、圖像、音樂、程式碼）
代表模型：GPT（文字）、DALL-E/Stable Diffusion（圖像）、Sora（影片）
名詞說明
LLM大型語言模型，以海量文字訓練，能生成、理解文字
TokenLLM 處理文字的基本單位，約 0.75 個英文單字
Context Window模型一次能處理的最大 Token 數量（越大越好）
Temperature控制輸出隨機性：0 = 確定、1 = 創意、>1 = 混亂
幻覺 Hallucination模型自信地輸出不正確資訊，是 LLM 主要缺陷
多模態 AI同時處理文字、圖像、音訊等多種資料類型
⬥ AI Agent（115 年高頻新考點）
定義：能自主感知環境、規劃行動、執行任務的 AI 系統
架構：感知（輸入）→ 記憶（上下文）→ 規劃（推理）→ 行動（執行）
例子：自動搜尋、寫報告、發郵件的 AI 助理

Prompt 提示工程與 RAG
L12201 · L12202 · 您的優勢區
必考
▶
技術說明特點
Zero-shot不提供範例，直接下指令快速，效果普通
Few-shot提供 2–5 個示例讓模型學習格式效果好，成本略高
Chain-of-Thought要求「一步一步思考」（Let's think step by step）提升複雜推理準確率
Tree-of-Thought探索多條推理路徑後選最佳（CoT 升級版）解複雜問題
RAG先檢索相關文件，再讓模型根據文件生成答案降低幻覺、知識可更新
⬥ 好 Prompt 的四要素
1. 角色設定：「你是一位資深 HR 顧問...」
2. 任務明確：具體說明要做什麼
3. 格式指定：「請用條列方式輸出」
4. 範例提供：Few-shot 效果更穩定
⬥ RAG vs Fine-tuning 選擇
RAG：知識會更新、需要引用來源、預算有限 → 選 RAG
Fine-tuning：需要特定風格/語氣、任務固定且高頻、有大量標註資料 → 選 Fine-tuning

No Code / Low Code 與 AI 導入規劃
L12301 · L12302 · L12303
必考
▶
No CodeLow Code傳統開發
技術門檻無需程式少量程式完整程式
開發速度最快快慢
客製化低中高
擴展性低中高
代表工具Zapier、MakePower AppsPython/JS
⬥ AI 導入評估五步驟
1. 需求分析：確認業務痛點，AI 是否為最佳解
2. 可行性評估：資料品質、技術成熟度、法規合規
3. POC 驗證：小規模試驗，驗證假設
4. 成本效益分析（ROI）：量化節省人力 + 質化決策品質提升
5. 風險管理：技術/資料/法規/組織文化風險識別與緩解
⬥ HR 背景加分：AI 導入失敗的主因
考試情境題答「AI 導入為何失敗」，答案通常是：
✅ 組織文化阻力、缺乏員工培訓、資料品質不足
❌ 不是「技術不夠先進」或「預算不足」（這些是次因）
⬥ 廠商鎖定（Vendor Lock-in）風險（115-2 新考點）
使用封閉式平台，資料以專有格式儲存、匯出受限、無標準 API → 未來想換系統卻被資料「綁住」
HR 比喻：就像員工所有人脈都在某個平台，離職後帶不走——換系統的成本極高
評估平台時應優先確認：開放 API、標準格式匯出、無廠商綁定條款
⬥ Webhook vs 輪詢（Polling）（115-2 新考點）
輪詢：系統定時主動去問「有新資料嗎？」→ 浪費資源，可能有延遲
Webhook：事先登記，有事件發生時系統主動推送通知 → 即時、省資源
HR 比喻：輪詢=每十分鐘打電話問「錄取了嗎？」；Webhook=錄取時 HR 自動傳訊通知你
口訣：Webhook = 有事叫我，Polling = 我定時去問
⬥ No-Code 工作流程核心元件
條件分支（Branching/Router）：依多個條件判斷流程走向（如退費金額 >1萬 AND 退費意圖 → 通知主管）
迭代/迴圈（Iterator）：逐筆處理多筆資料
陣列聚合器（Aggregator）：將多筆資料整合後再處理
多輪對話記憶：LLM 本身沒有記憶，每次 API 請求必須把完整對話歷史打包進 Context 一起送出
⬥ Vibe Coding vs Low-Code（115-2 新考點）
Vibe Coding：用自然語言指示 AI 生成程式碼，彈性高，但品質依賴 Prompt 精準度，需要開發者審閱
Low-Code：預設元件拖拉組合，快速穩定，但功能上限受平台支援範疇限制
兩者都不是「完全不需人工」——Vibe Coding 需審閱程式碼，Low-Code 需設定元件邏輯

AI 導入策略、變革管理與組織文化
L12302 · HR 背景最有優勢的題型
必考
▶
⬥ HR 視角：AI 導入失敗的真正原因
考試題目問「AI 導入失敗最主要原因」→ 答案永遠是人的問題，不是技術問題
✅ 組織文化阻力、員工缺乏培訓、資料品質不足
❌ 不是「技術不夠先進」或「預算不足」（這些是次要因素）
⬥ 變革管理（Change Management）核心策略
Quick Wins（快速贏得）：先在最容易成功的場景取得具體成效，建立員工信心，再逐步擴大
分層培訓：主管學 AI 策略與治理；業務人員學工具與 Prompt；技術人員學開發與部署
充分溝通：說明 AI 是「輔助」而非「取代」，降低員工焦慮
用戶接受度是 AI ROI 的乘數：再好的工具沒人用 = 零效益
AI ROI 類型範例特點
直接效益每月節省 40 小時人工、錯誤率降低 30%可量化，容易說服管理層
間接效益員工滿意度↑、創新能力↑、離職率↓難量化，但長期影響更深遠
⬥ AI 治理框架與倫理委員會
AI 使用政策應包含：允許/禁止使用場景、資料安全限制（哪些不能輸入 AI）、輸出審核要求、版權聲明、培訓要求
AI 倫理委員會：跨部門組成（法務/HR/IT/業務），事前審查高風險 AI 應用，不是事後救火
AI 路線圖：優先順序 + 資源規劃 + 人才培育 + 資料治理 + 風險管控，是跨部門策略文件
⬥ HR 高風險 AI 應用（需人工審核）
高風險（必須人類把關）：完全自動化決定升遷/解雇/招募篩選
低風險（可自動採納）：撰寫職缺說明、生成培訓材料、電子郵件分類
口訣：影響人的命運就要人審核；例行作業可讓 AI 自動

生成式 AI 進階：向量資料庫、System Prompt 與 Multi-Agent
115 年新增高頻考點 · RAG 延伸知識
必考
▶
⬥ 向量資料庫（Vector Database）在 RAG 中的角色
RAG 流程：文件切片 → Embedding 模型轉為向量 → 存入向量資料庫 → 用戶提問也轉向量 → 語意搜尋找最相關片段 → 送給 LLM 生成答案
語意搜尋 vs 關鍵字搜尋：語意搜尋理解「意思」，查「薪資」也能找到「薪酬/報酬/待遇」的文件；關鍵字搜尋只找字面完全符合的
⬥ RAG 檢索品質優化
分塊策略（Chunking）：文件切太大=冗餘，切太小=語境不足，要找到平衡點
更好的 Embedding 模型：語意理解更準確，找到的片段更相關
查詢增強（Query Enhancement）：把用戶問題改寫得更精準，匹配文件用語
→ 這些改善比換更大的 LLM 更有效也更省錢
⬥ System Prompt（系統提示）
對話開始前設定 AI 的角色、規則、格式、限制，對整個對話持續有效
HR 比喻：就像員工到職前的「工作說明書」，設定好之後員工知道自己的職責和不能做的事
企業用途：設定客服 AI 的回答範圍、語氣、禁止話題
概念說明
Multi-Agent 系統多個 AI Agent 分工協作（一個搜尋、一個分析、一個寫作），完成單一 Agent 做不到的複雜任務；最大挑戰：各 Agent 的 Context 同步，避免基於不同資訊做出矛盾決策
幻覺的根本原因LLM 是「預測下一個最可能的詞」的統計模型，不是查詢事實資料庫，不確定時會「猜」
持續學習 / 災難性遺忘模型學新任務時覆蓋舊任務的參數，導致「忘記」舊能力
差分隱私在統計結果加入數學校準的雜訊，讓個別資料無法被識別，同時保留整體統計有用性
CLIP 模型用圖像與文字配對資料訓練，學習圖文語意對應；用餘弦相似度衡量文字和圖片向量的相似程度，是「文字搜圖」的核心技術
擴散模型（Diffusion Model）從純噪聲開始，透過多步驟「去噪」逐步生成圖像/影片；Sora 等影片生成模型的核心機制，與 GAN「一次生成」不同
訓練資料記憶洩漏模型若以含真實個資的資料訓練，可能在特定對話情境下「回想」並輸出敏感資訊；訓練資料必須嚴格去識別化
間接 Prompt Injection攻擊者將惡意指令藏在模型會讀取的外部內容（網頁、文件）中，LLM 處理時被間接控制；直接注入=用戶輸入，間接注入=外部資料
⬥ MCP（Model Context Protocol）（115-2 新考點）
讓模型在推理時即時存取外部文件/資料庫的協定，不需重新訓練
使用場景：AI Coding 助手讀取最新 API 文件、企業知識助理存取最新規範
與 RAG 的差異：RAG 是預先建立向量庫再檢索，MCP 是推理時直接連接外部工具
⬥ Extended Thinking（延伸推理）Token 預算
推理型 LLM 在給出答案前可進行深度思考，思考過程消耗額外 Token（計費）
動態 Token 預算：簡單問題少給 Token（省成本），複雜問題多給 Token（保品質）
Temperature 控制「輸出多樣性」；Token 預算控制「思考深度」——兩者不同

Agentic AI：ReAct、Function Calling 與 No-Code 工具整合
115-2 大量新增考點 · 科目二必考
必考
▶
⬥ AI Agent vs 傳統自動化的核心差異
傳統自動化：照固定劇本執行，遇到意外就停擺
AI Agent：能動態規劃步驟、調用工具、根據中間結果調整策略
HR 比喻：傳統自動化=按 SOP 執行的助理；AI Agent=能自主判斷、遇到問題自己想辦法的專員
Agent 四層架構：感知（輸入）→ 記憶（Context）→ 規劃（Task Planner）→ 行動（Tool Executor）
⬥ ReAct 架構（Reason and Act）
讓 AI 在執行任務時進行「推理→行動→觀察→再推理」的動態循環
適合場景：任務步驟不確定、需根據中間結果調整方向（如：分析融資狀況，AI 自行判斷資訊是否足夠）
不適合場景：固定流程任務（批次翻譯 PDF、依規則判斷退貨資格）→ 不需要 ReAct
口訣：動態決策用 ReAct；固定流程不需要
⬥ Function Calling（函數呼叫）
LLM 理解用戶的自然語言需求，自動轉換為可執行的 API 呼叫參數
例：用語音說「幫我訂明天下午三點的外送」→ LLM 解析意圖 → 自動填入 API 所需參數（時間、品項）→ 呼叫外送 API
這是 AI 系統連接外部工具（地圖、資料庫、ERP）的關鍵技術
概念說明
Task Planner（任務規劃器）Agentic AI 的核心元件，負責把高層目標拆解為子任務並決定執行順序
Tool Executor（工具執行器）根據 Task Planner 的指示執行具體操作（搜尋、計算、寫檔案）
Solution Graph把問題解決步驟表示為有向無環圖（DAG），節點=子任務，連結=依賴關係；LLM 能推理建構此圖作為行動計畫
Agentic Coding（如 Claude Code）不只是「建議程式碼」，而是能直接讀寫專案檔案、執行終端機指令、提交 commit；生成的程式碼仍需人工審查
Memory as Context（MAC）把長期記憶整理後加入每次對話的 Context，讓模型能「記住」過去的重要資訊；Titans 架構採用此機制
RFT（強化微調）vs SFTSFT（監督式微調）需大量標籤資料；RFT 透過獎懲機制引導，適合標籤稀缺場景（如特定醫療任務）
GRPO強化學習優化方法，特別適合有明確對/錯答案的推理任務（數學、邏輯謎題）；DeepSeek-R1 採用類似方法
CoT 衍生技術ReAct、Tree-of-Thought、Self-consistency 都是 CoT 的延伸；Zero-shot Learning 是另一個獨立概念，不是 CoT 衍生
⬥ Prompt Injection 多層防禦（115-2 考點）
單一防線容易被繞過，需採用多層防禦（Defense in Depth）：
① 輸入驗證（過濾可疑指令）
② 輸出監控（偵測不當內容）
③ System Prompt 保護（嚴格限制模型行為範疇）
④ 最小權限原則（AI 只能存取它需要的資源）
直接注入=用戶輸入嵌入惡意指令；間接注入=惡意指令藏在外部文件/網頁
⬥ HR 視角：Agentic AI 的高/低風險場景
✅ 可讓 AI 自主執行：低風險例行任務（電郵分類、資料彙整、初步排程）
⚠️ 需人類最終確認：高風險決策（人事決定、合約簽署、資金調撥）
口訣：影響人的命運就要人審核；例行作業讓 AI 自動
⬥ AutoSkill 框架 / MMLU 評測基準
AutoSkill：從 AI 與環境的重複互動中，自動提取可重複使用的「技能模組」；像是 AI 版的「建立 SOP」，讓 AI 積累並複用技能
MMLU（多領域多任務語言理解）：跨法律、醫學、數學、歷史等多學科評測 LLM 的泛化推理能力，測的是「舉一反三」，不是死記硬背
